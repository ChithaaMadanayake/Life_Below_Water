Home Page

Technical Description

Created an engaging homepage with a video background. Used <video> for the video background, with autoplay, loop, and muted attributes to enhance visual appeal. Applied CSS animations for smooth transitions and added a parallax effect for a dynamic feel.

Accessibility

Ensured the site is accessible by using semantic HTML elements like <header>, <main>, and <footer> to improve screen reader compatibility. Provided descriptive alt text for all images, ensuring that visually impaired users can understand the content through assistive technologies.

Table Page

Technical Description

Designed a structured table page for better organization. Designed using table elements, styled for clarity and easy reading. Used nth-child selectors to alternate row colors for better readability.

Accessibility

Implemented keyboard navigability for the navigation bar by ensuring all interactive elements, such as links and buttons, are reachable and focusable through keyboard navigation. I also included clear focus styles to help users visually track which element they are interacting with.

Content Page

Technical Description

Created an informative content page with well-structured data. Structured data using grid and flexbox to create a responsive layout. Applied box-shadow and border-radius for a modern look.

Accessibility

Ensured that the color contrast between the text and background meets the Web Content Accessibility Guidelines (WCAG) for readability by users with visual impairments. Additionally, I used ARIA (Accessible Rich Internet Applications) landmarks and roles to provide additional context for users with screen readers. By integrating these best practices, the website ensures that all users, regardless of ability, can navigate and access the content effectively, fostering a more inclusive user experience.

Challenges and Lessons Learned

Some challenges included ensuring smooth responsiveness across devices and implementing animations effectively. I had to carefully balance performance and design, especially when optimizing the background video for faster load times without sacrificing quality. Additionally, creating a navigation bar that remained functional across different screen sizes and browsers required extra attention to media queries and CSS flex properties.

Another challenge was debugging complex CSS issues, such as fixing layout shifts and resolving conflicts between different styles. I learned how to use browser developer tools more efficiently to inspect elements, track down errors, and experiment with fixes in real-time. I also faced challenges with ensuring accessibility across all devices, making sure that interactive elements were fully keyboard-navigable and met web accessibility standards.

Overall, I learned the importance of balancing aesthetics with functionality and performance. The process taught me how to optimize resources, ensure faster page load times, and design for a diverse range of devices and users. This project also deepened my understanding of CSS layout techniques, animations, and accessibility best practices.

Compliance

The page follows standard web accessibility guidelines, including the Web Content Accessibility Guidelines (WCAG) 2.1, ensuring that it meets accessibility standards for people with disabilities. This includes considerations for text contrast, keyboard navigation, and screen reader compatibility.

Additionally, the website complies with Janet regulations, which govern the use of digital services in the UK. It ensures that the site meets the security, data privacy, and accessibility requirements set out by Janet for higher education and research institutions.

The site has been designed to provide an inclusive and accessible user experience while maintaining compliance with relevant legal and regulatory standards to ensure equal access for all users.
