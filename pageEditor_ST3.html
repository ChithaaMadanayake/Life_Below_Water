Feedback

Technical Description

The Feedback page was built using HTML5 forms with input types like text, email, radio buttons, and textarea to collect user input. CSS was used to style the form elements, ensuring they were visually aligned and easy to interact with. Important tags include <form>, <input>, <label>, and <textarea>. CSS properties like padding, margin, border, and background-color were used to enhance the design and improve readability.

Accessibility

Labels were used with every input field to help screen readers. Placeholder texts were avoided for critical instructions. The form layout ensures a logical tab order, making it keyboard-friendly. Color contrast was checked to be readable for users with visual impairments.

Team Page

Technical Description

The Team page uses a combination of HTML sections, images, and grid-based layout to introduce team members. The <section> and <article> tags provide structure, while CSS Flexbox aligns the cards side by side responsively. Styling includes border-radius for smooth corners, box-shadow for depth, and consistent font usage for a clean look.

Accessibility

Alt text is provided for all images, helping screen readers describe the visuals. Clear heading levels help assistive tools navigate the content. The page is responsive, ensuring accessibility on all screen sizes and devices.

Content Page

Technical Description

This page was structured using semantic HTML elements like <header>, <main>, <section>, and <footer> for clear content separation. CSS properties like margin, padding, background-color, and text-align were used for spacing and alignment. A consistent color theme and font family were used for visual harmony.

Accessibility

Text is kept readable with high contrast colors and larger font sizes. Proper use of semantic tags helps screen readers understand the layout. The page structure supports keyboard navigation and adapts well on smaller devices.

Challenges and Lessons Learned

One of the challenges was aligning elements correctly using Flexbox, especially when making the layout responsive. Another was ensuring all form fields were both user-friendly and accessible. Through trial and error, I learned to use semantic HTML and CSS best practices to build clean and structured pages.

Compliance

The pages follow Janet web publishing regulations by using valid HTML5 structure, accessible design practices, and responsive layouts. Forms include input validation, and all content is original and cited. CSS is cleanly separated from HTML, and external links are included for further validation.
