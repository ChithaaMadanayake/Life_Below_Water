Splash Page

Technical Description

Html Structure

DOCTYPE and HTML Tag:

  • <!DOCTYPE html> declares the document type and version of HTML (HTML5).
  • <html lang="en"> specifies the language of the document (English), which is important for accessibility and SEO.

Meta Tags:

  • <meta charset="UTF-8"> ensures the document uses UTF-8 character encoding, which supports a wide range of characters.
  • <meta name="viewport" content="width=device-width, initial-scale=1"> makes the page responsive by setting the viewport width to the device's width and initial zoom level to 1.
  • <meta http-equiv="refresh" content="4; url=home.html"> automatically redirects the user to home.html after 4 seconds, which is useful for splash screens.

Title Tag:

  • <title>Splash Screen</title> sets the title of the webpage, which appears in the browser tab and is important for SEO.

Body Content:

  • The <body> contains the main content of the page, including the splash screen elements.
  • <div class="splash-container"> is a container for the splash screen content, using flexbox to center the content both horizontally and vertically.
  • <h1>, <p>, and <h2> tags are used for headings and text, with specific styling applied to create a visually appealing splash screen.
  • <div class="fish1"> and <div class="fish2"> contain animated fish images that move across the screen, adding a dynamic element to the design.
  • <div class="members"> contains a list of student names, styled to appear as inline blocks with a light background and shadow effect.

CSS Styling

Body Styling:

  • background: url("life_underwater.jpg"); sets a background image that covers the entire screen without repeating.
  • display: flex; justify-content: center; align-items: center; uses flexbox to center the content both horizontally and vertically.
  • animation: fadeOut 1s ease-in-out 6s forwards; applies a fade-out animation to the body after 6 seconds, creating a smooth transition before the page redirects.

Splash Container Styling:

  • text-align: center; centers the text inside the container.
  • animation: fadeOut 1s ease-in-out 3s forwards; applies a fade-out animation to the container after 3 seconds.

Text Styling:

  • font-family, font-size, color, and text-shadow are used to style the headings and paragraphs, making them visually appealing and readable.
  • opacity: 0; initially hides the text, and animation: fadeIn gradually makes it visible.

Fish Animation:

  • .fish1 img and .fish2 img are styled with position: absolute; and animation properties to create the effect of fish swimming across the screen.
  • @keyframes moveRight and @keyframes moveLeft define the animations for the fish moving from left to right and right to left, respectively.

Members List Styling:

  • .members ul removes default list styling and centers the list items.
  • .members li styles each list item with a light background, rounded corners, and a shadow effect, making them stand out.

Accessibility

Semantic HTML:

  • The use of semantic HTML tags like <h1>, <p>, and <ul> helps screen readers understand the structure and content of the page, improving accessibility for visually impaired users.

Alt Text for Images:

  • The alt attribute in the <img> tags (alt="Animated fish swimming across the screen") provides alternative text for the fish images. This is crucial for screen readers to describe the images to users who cannot see them.

Color Contrast:

  • The text color (color: white; for headings and color: black; for paragraphs) is chosen to ensure good contrast against the background, making the text readable for users with visual impairments.

Keyboard Navigation:

  • The page is designed to be navigable via keyboard, as there are no interactive elements that would require mouse-specific actions. The automatic redirect after 4 seconds ensures that users who rely on keyboard navigation are not stuck on the splash screen indefinitely.

Responsive Design:

  • The use of the viewport meta tag and flexbox ensures that the page is responsive and accessible on various devices, including mobile phones and tablets.

Content Page

Technical Description

Semantic HTML Tags:

  • <section>: Used to divide the page into logical sections like the introduction, blog posts, and image gallery.
  • <header>: Contains the sidebar navigation and introductory content.
  • <main>: Wraps the main content of the page, including the blog posts and image gallery.
  • <footer>: Contains the "Go to Top" button for easy navigation.

Sidebar Navigation:

  • <div class="top-destination">: A fixed sidebar on the left side of the page, providing quick navigation links to different sections of the page.
  • <div class="top3">: Each navigation item in the sidebar, containing an icon and a link to a specific section.

Blog Posts:

  • <div class="blog-container">: A flex container that holds individual blog posts.
  • <div class="blog-box">: Each blog post contains an image, title, date, and description. The posts are styled to be visually appealing and responsive.

Image Gallery:

  • <div class="image-container">: A flex container that holds sets of images. Each image box is styled with rounded corners, shadows, and hover effects for interactivity.
  • <div class="image-box">: Each image box contains a background image and a caption, styled to be visually appealing and responsive.

CSS Styling:

  • flexbox: Used extensively for layout, ensuring the page is responsive and adapts to different screen sizes.
  • box-shadow: Adds subtle shadows to cards and images, giving them a lifted appearance.
  • transition: Used for smooth hover effects on cards, images, and links, enhancing user interaction.
  • background: linear-gradient: Creates a gradient background for the page, adding visual depth.

Challenges and Lessons Learned

Hover Effects and Transitions:

Challenges:

Implementing smooth hover effects on cards and buttons without causing performance issues.

Solutions:

Used CSS transitions (e.g., transition: transform 0.3s ease) to create smooth animations for card hover effects.

Lessons Learned:

Simple CSS transitions can improve user interaction without affecting the performance.

Dropdown and Popup Functionality:

Challenges:

Implementing the sorting dropdown on the Volunteer Page and the reviews popup without JavaScript.

Solutions:

Used CSS-only techniques (:target for the popup) to create interactive elements.

Lessons Learned:

CSS can handle many interactive features without needing JavaScript, but it has its own limitations.

Read More Button (Volunteer Page):

Challenges:

Implementing the "Read More" functionality for reviews without JavaScript.

Solutions:

Used a CSS checkbox hack without using JavaScript.
